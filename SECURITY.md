# Security Policy

## Reporting a security issue

Do not publish credentials, customer information, internal hostnames, or sensitive diagnostic output in a public issue.

Report suspected security problems privately to the repository owner. Include the affected function, Windows version, reproduction steps, and the minimum diagnostic detail required to investigate.

## Operational guidance

- Review code before running it with administrator privileges.
- Test disruptive repair actions on a non-production device first.
- Back up important data and configuration before system-level changes.
- Validate downloaded tools and follow organisational application-control policies.
- Remove logs and reports that contain environment-specific information before sharing them.
